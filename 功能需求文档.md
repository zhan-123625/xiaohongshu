# 仿小红书APP功能需求文档

## 1. 引言

### 1.1 文档目的
本文档旨在明确“仿小红书”APP的功能需求，为开发团队、测试团队及项目管理人员提供详细的功能描述和实施依据，确保产品开发符合预期的业务目标。

### 1.2 产品背景
本项目旨在打造一个年轻人的生活方式分享平台，通过图文、短视频等形式，让用户记录生活、分享经验、发现美好。核心机制为“UGC内容生产 + 算法推荐 + 社区互动”。

## 2. 产品概述

### 2.1 产品定位
一个集内容分享、社交互动、兴趣探索于一体的移动端社区应用。

### 2.2 核心业务流程
1.  **内容消费**：用户浏览推荐流 -> 点击详情 -> 互动（点赞/收藏/评论） -> 关注作者。
2.  **内容生产**：用户点击发布 -> 选择图片/视频 -> 编辑滤镜/贴纸 -> 添加标题/正文/标签 -> 发布成功。

## 3. 用户角色

| 角色 | 描述 | 主要权限 |
| :--- | :--- | :--- |
| **普通用户** | 平台的主要使用者，既是浏览者也是创作者 | 浏览内容、发布笔记、互动评论、关注用户、私信聊天 |
| **管理员** | 平台运营与维护人员 | 内容审核、用户管理、标签管理、推荐干预 |

## 4. 功能模块详解

### 4.1 注册与登录模块
*   **手机号注册/登录**：支持手机号+验证码方式登录/注册。
*   **第三方登录**：支持微信、QQ、微博一键授权登录。
*   **完善资料**：新用户注册后引导填写昵称、性别、生日、感兴趣的领域（用于冷启动推荐）。

### 4.2 首页/发现模块（核心）
*   **关注页**：展示用户已关注的博主发布的最新笔记，按时间倒序排列。
*   **发现页（推荐流）**：
    *   采用双列瀑布流布局。
    *   基于用户兴趣标签和浏览历史进行算法推荐。
    *   支持下拉刷新、上拉加载更多。
*   **同城页**：基于LBS地理位置，展示附近用户发布的笔记。

### 4.3 内容发布模块（核心）
*   **媒体选择**：支持从相册选择多张图片（最多9张）或上传短视频。
*   **图片编辑**：提供滤镜、裁剪、贴纸、标签（标记品牌、地点、用户）功能。
*   **内容编辑**：
    *   填写标题（限制字数）。
    *   填写正文内容（支持表情、话题#、艾特@）。
    *   选择参与的话题/活动。
    *   添加地理位置。
*   **草稿箱**：支持保存未发布的笔记至本地草稿箱。

### 4.4 笔记详情与互动模块
*   **笔记展示**：
    *   图片轮播/视频播放。
    *   展示用户信息、关注按钮。
    *   展示标题、正文、发布时间、地点。
*   **互动功能**：
    *   **点赞**：双击图片或点击爱心图标点赞。
    *   **收藏**：点击星星图标收藏，支持加入自定义收藏夹（专辑）。
    *   **评论**：支持一级评论和二级回复，支持评论点赞。
    *   **分享**：生成分享海报、分享链接至第三方平台。

### 4.5 搜索模块
*   **关键词搜索**：支持搜索笔记、用户、话题。
*   **热门搜索**：展示当前平台热度最高的关键词。
*   **搜索历史**：记录用户搜索历史，支持一键清空。
*   **搜索筛选**：支持按综合、最新、最热进行排序筛选。

### 4.6 个人中心模块
*   **个人主页**：展示头像、昵称、简介、获赞数、关注数、粉丝数。
    *   **关注/粉丝列表**：点击关注数或粉丝数进入列表，支持在列表中直接关注/取消关注、发私信。
    *   **粉丝管理**：在粉丝列表中支持拉黑用户。
*   **我的笔记**：展示自己发布的笔记列表。
*   **我的收藏**：展示收藏的笔记和专辑。
*   **我的赞过**：展示点赞过的笔记列表（可设为隐私）。
*   **设置**：账号安全、隐私设置、消息通知设置、退出登录。

### 4.7 消息通知模块
*   **消息首页**：
    *   **顶部入口**：聚合展示“赞和收藏”、“新增关注”、“评论和@”三类互动通知，有红点提示。
    *   **会话列表**：展示私信聊天记录，按时间倒序排列，显示最后一条消息内容和时间。
*   **互动消息详情**：
    *   **赞和收藏**：列表展示谁赞了/收藏了哪篇笔记，点击可跳转至笔记详情。
    *   **新增关注**：列表展示新增粉丝，支持回粉。
*   **私信聊天**：
    *   支持一对一即时文本聊天。
    *   支持从个人主页或关注/粉丝列表发起私信。
    *   聊天界面区分发送方（右侧）和接收方（左侧）样式。
*   **系统通知**：平台公告、审核结果通知。

## 5. 非功能需求

### 5.1 性能需求
*   **加载速度**：首页瀑布流图片加载需进行懒加载优化，首屏加载时间不超过1.5秒。
*   **并发支持**：支持高并发访问，保证推荐服务的稳定性。

### 5.2 安全需求
*   **数据安全**：用户密码加密存储，敏感数据传输加密。
*   **内容安全**：接入文本和图片自动审核机制，过滤涉黄、涉政、暴力等违规内容。

### 5.3 兼容性
*   支持 iOS 和 Android 主流版本及机型。
